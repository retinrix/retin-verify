Problem Restatement
Despite improvements in v5 (front↔back errors reduced from 255 to 13), front accuracy remains low (72%) while back accuracy (80%) and no‑card (97%) are higher. The symmetric front‑back confusion is now small, but the model still fails on a significant fraction of real front images. This indicates the model does not consistently recognize the distinctive features of the front side.

Why Front Is Still Struggling
Synthetic data bias – Your synthetic front images are generated by extracting backgrounds, adding text fields, and inserting an ICAO photo. Real front images have glare, shadows, angle variations, and natural texture that the synthetic pipeline does not reproduce. The model learned a synthetic‑specific representation that doesn’t generalize.

Class‑imbalanced training signal – Even with real‑weighted sampling, the model sees many synthetic examples, and the real front examples are limited. The 2‑class real‑only model achieved near‑perfect performance, proving that real data alone is sufficient.

Third class “no‑card” dilutes capacity – The model must also distinguish CNIE from other documents, adding complexity. However, your confusion matrix shows that no‑card is rarely mistaken for front/back. The real bottleneck is the front‑back discrimination when the input is indeed a CNIE.

Lack of fine‑grained discriminative features – Front and back share similar backgrounds, text fonts, and overall layout. The model may rely on subtle cues (e.g., presence of a photo vs. fingerprint). Synthetic data may not capture these cues consistently, leading to under‑learning on real front images.

A New Efficient Approach
Instead of a single 3‑class model, decouple the problem into two independent binary tasks. This avoids the negative transfer from synthetic data to the front‑back discrimination and allows you to train each component on its optimal data.

1. Stage 1: Document Type Detector (CNIE vs. no‑card)
Goal: Determine whether the image contains a CNIE card (any side) or another document.

Data: Use all real CNIE images (front + back) as positive class, and all no‑card samples (credit cards, passports, etc.) as negative.

Training: Since the task is coarse, you can use a lightweight model (e.g., MobileNetV3) trained on real data + heavy augmentation. This classifier will have high accuracy because the visual differences between CNIE and other documents are significant.

Output: If “no‑card”, stop; else pass to Stage 2.

2. Stage 2: Front/Back Classifier (CNIE only)
Goal: Distinguish front from back, only on images already classified as CNIE.

Data: Use only real front and back images – no synthetic data. You already have a set of real images (training set) plus feedback images (135 front misclassified as back, 120 back misclassified as front). These are perfect hard negatives for retraining.

Training:

Model: EfficientNet‑B0 (or even smaller) pretrained on ImageNet.

Augmentations: Add glare simulation, random perspective warp, motion blur, and random occlusion to mimic real conditions.

Regularization: Use MixUp and CutMix (explained earlier) to improve robustness.

Loss: Standard cross‑entropy with class weights balanced (front/back roughly equal in your collection).

Expected accuracy: Based on your earlier 2‑class experiment, you can achieve >98% on real images, especially after incorporating the feedback examples.

3. Optional: Region‑of‑Interest (ROI) Cues
Front CNIE has a photo area; back has a fingerprint area and MRZ. You can detect these regions using a small object detector (e.g., YOLO) and feed the ROI features as an auxiliary input to the classifier. This forces the model to focus on the most discriminative parts. Since the positions are relatively fixed, a simple template‑based approach may also work.

Why This Approach Is Efficient
Minimal new data needed – You already have all the real images and feedback examples.

Fast training – Each stage uses a small, focused dataset. The binary classifiers converge quickly.

Eliminates synthetic bias – The front‑back classifier uses only real images, so it learns the true distribution.

Modular – You can improve each stage independently. If you later collect more real data, you can fine‑tune only the relevant component.

Easy to deploy – Inference is two simple forward passes; latency is negligible.

Implementation Steps
Prepare datasets

Stage 1 (CNIE vs. no‑card):

Positive: all real front + back images (from train/ and val/ folders).

Negative: all no‑card images (from your 107 feedback no‑card samples + any other non‑CNIE documents).

Stage 2 (front vs. back):

Front: all real front images + feedback front images (the ones misclassified as back).

Back: all real back images + feedback back images.

Train Stage 1 (CNIE detector)

Use a small model (e.g., MobileNetV3‑small) with binary output.

Use the same augmentation as in v4/v5 (rotation, color jitter, etc.).

Train until validation accuracy plateaus (likely 10‑20 epochs).

Train Stage 2 (front/back)

Use EfficientNet‑B0 with differential learning rates (backbone 1e‑5, classifier 1e‑3).

Augmentation: add glare, perspective warp, blur, and occlusion. Use MixUp/CutMix (set α=0.2 for MixUp, α=1.0 for CutMix).

Use early stopping (patience 5) and save the best model based on validation accuracy on real images.

Since you have only real data, overfitting is less likely; but still use dropout (0.5) and weight decay.

Evaluate on a held‑out test set (real images not seen during training).

Measure accuracy of the cascade: first pass through CNIE detector; if predicted CNIE, then pass to front/back classifier.

Compute per‑class accuracy for front, back, and no‑card.

Iterate – If front accuracy still lags, collect more challenging front images (e.g., extreme angles, glare) and add them to Stage 2 training.

Expected Outcomes
Front accuracy should rise from 72% to ≥95% on real test images.

Back accuracy should similarly improve to ≥95%.

No‑card accuracy remains high (already 97%).

Overall system accuracy becomes robust and maintainable.

Why Not Use the v5 Model as a Starting Point?
The v5 model already has decent front‑back separation (only 13 errors) but low front accuracy. You could try fine‑tuning v5 on real data only (especially the feedback images) while freezing early layers. However, the synthetic bias may still linger. A fresh start on real data (as proposed) is cleaner and faster.
